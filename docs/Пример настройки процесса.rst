Практические примеры
======================

Раздел содержит пошаговые примеры создания приложений в Citeck — от простых сценариев до комплексных бизнес-процессов. Каждый пример демонстрирует реальный рабочий процесс и знакомит с ключевыми инструментами платформы.

**Основные примеры**

:ref:`Закупка оборудования <sample_request>` *(начальный уровень)*
   Автоматизация процесса заявки на закупку. Оптимальная отправная точка для знакомства с платформой: тип данных, форма, журнал и BPMN-процесс.

:ref:`Согласование офферов <sample_offer>` *(продвинутый уровень)*
   Автоматизация HR-процесса найма: от создания оффера до финального решения. Комплексный пример с несколькими участниками и ветвлением логики.

**Дополнительные примеры**

- :ref:`Создание пользователей <demo_user>` и :ref:`рабочего пространства <demo_ws>` — подготовка среды перед работой с примерами;
- :ref:`Расширение базового процесса <sample_request_complicated>` — справочник, журнал, пользовательская форма;
- :ref:`Динамические роли через DMN <dynamic_role_dmn>` — назначение исполнителя по условию;
- :ref:`Подключение DMN-таблицы <sample_request_dmn>` — автоматическое принятие решений;
- :ref:`Добавление событий BPMN <sample_request_events>` — таймеры и уведомления;
- :ref:`Создание процесса с помощью ИИ-ассистента <sample_request_ai>` — альтернативный способ разработки процессов;
- :ref:`Настройка дашборда и графической статистики <sample_dashboard>` — визуализация данных;
- :ref:`Добавление канбан-доски <sample_kanban_board>` — управление задачами в виде доски;
- :ref:`Упаковка готового приложения <app_example>` — экспорт и деплой на другой стенд.

Приложение в Citeck строится из взаимосвязанных артефактов. Артефакты создаются последовательно:

.. list-table::
   :widths: 5 30
   :header-rows: 1
   :class: tight-table

   * - Артефакт
     - Описание
   * - **Тип данных**
     - Основной артефакт системы, описывающий объект. Определяет метаданные, статусы жизненного цикла и роли. Связан с формой и журналом.
   * - **Форма**
     - Графическое представление объекта в виде набора UI-элементов для работы с данными. Элементы формы ссылаются на атрибуты типа данных.
   * - **Журнал**
     - Табличное представление объектов с возможностью настройки столбцов, фильтрации и управления записями. Столбцы соответствуют атрибутам объекта или вычисляются на их основе.
   * - **Бизнес-процесс**
     - Последовательность действий для получения результата, ценного для организации. Создаётся с помощью встроенного BPMN-конструктора.
   * - **Таблица решений (DMN)**
     - Правила принятия решений, описанные в табличном виде. Позволяет автоматически определять исход (например, выбирать согласующего) на основании заданных условий.

Готовые артефакты можно упаковать в приложение, экспортировать и разместить в системе контроля версий или задеплоить на другой стенд.
