Разработка
===========

.. important::

    Eсли система поднята локально без eis (keycloak), то используется **BASIC_AUTH_ACCESS** и при добавлении пользователя его нужно добавить в :ref:`docker-файл<ecos_proxy_app>`

Раздел содержит техническую документацию для разработчиков платформы Citeck. Здесь описаны объекты и сущности платформы, механизмы работы с данными, ключевые внутренние механизмы — транзакции, события, Web API. Отдельные статьи посвящены архитектуре микросервисов и техническому описанию реализованных функциональностей. Также представлены инструменты разработчика, настройка бэкенд-локализации, конфигурация AI-ассистента и практические how-to руководства. Критические изменения при обновлении платформы вынесены в отдельный раздел.
