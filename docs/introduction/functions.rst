Описание отдельных функциональностей
=====================================

В разделе описаны кросс-модульные функциональности платформы Citeck — инструменты, доступные в различных модулях системы независимо от конкретного бизнес-процесса или типа записи.

- :ref:`Клиент 360 <client_360>` — подход к работе с контрагентами: вся информация о клиенте собрана в едином месте — история взаимодействий, документы, задачи и связанные записи.
- :ref:`Предпросмотр карточки <preview>` — встроенный режим просмотра данных записи прямо из журнала, без открытия полной карточки.
- :ref:`Дерево категорий <category_tree>` — инструмент для иерархической организации записей журнала по произвольным категориям.
- :ref:`Редактирование в OnlyOffice <edit_only_office>` — открытие и редактирование файлов форматов MS Office и OpenDocument прямо в браузере без сторонних приложений.
- :ref:`Делегирование <delegation>` — механизм передачи задач и полномочий другим сотрудникам — на период отсутствия или на постоянной основе.
- :ref:`WYSIWYG-редактор <wysiwyg_editor>` — визуальный редактор для создания форматированного контента: текст, таблицы, изображения и формулы.
- :ref:`Шаблоны документов <doc_templates>` — автоматическая генерация файлов (docx и др.) на основе заранее подготовленных шаблонов и данных из карточек записей.
- :ref:`Сравнение версий <version_comparison>` — наглядное отображение различий между двумя версиями документа: добавленные, изменённые и удалённые фрагменты подсвечиваются в интерфейсе.
- :ref:`Действия в карточках <actions>` — набор встроенных действий для управления записью: редактирование свойств, печать, загрузка новой версии, создание поручений.
- :ref:`Рабочее расписание и производственный календарь <business-schedule>` — инструменты для управления рабочим временем: настройка графиков, праздников и нерабочих дней, используемых в бизнес-процессах.
- :ref:`Импорт данных <default_data_import>` — загрузка данных из файлов Excel по настроенному шаблону — в результате в системе автоматически создаются карточки записей выбранного типа.
- :ref:`Штрихкод в PDF <barcode_pdf>` — автоматическое формирование PDF-документа со встроенным штрихкодом для идентификации и отслеживания документов в бизнес-процессах.
- :ref:`Электронная подпись (ЭЦП) <esign>` — подписание документов с использованием усиленной квалифицированной электронной подписи (УКЭП), обеспечивающей юридическую значимость.
